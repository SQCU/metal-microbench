A FastAPI bridge in front of a Gemma-4-A4B inference engine running on Metal / M5 Max. OpenAI-compatible REST at /v1/*; demo clients below. Every client is the same HTTP API you'd get from any chat endpoint — they differ only in how they exercise the engine's paged-attention + prefix-cache + batched-decode primitives.

tetraplex /tetraplex
Four concurrent chat streams, bandwidth chart + per-session KV tenancy strip. Original demo — exercises batch-of-4 AR decode against a shared model. Optional MODE=multimodal renders an amongus frame into each pane.
batched AR decode kv tenancy viz multimodal
image labeler /labeler
Drop PNGs (or load the bundled amongus frames), hand the engine a schema prompt, watch 4 live-decoding ribbons drain the queue. Downloadable JSONL of results. Shows vision tower cache, shared schema-prefix KV pages, and scheduler coalescing work-by-type.
vision cache shared prefix multi-slot softs prefill
loom /loom
Tree-of-thoughts. Fork N branches from a shared root, each biased by a seed suffix. Click any finished node to fork further from there. SVG threads connect parent to child. Watch the KV tenancy climb as sibling branches share their ancestor-path pages.

prompt A/B harness /compare
One shared system prompt, K (2-8) user-prompt variants, run concurrently with outputs side-by-side. The system prompt's KV pages are content-hashed and shared across every variant's session — the shared-pages counter in the footer climbs visibly as soon as you hit run.

REST endpoints

GET/health engine readiness, model name, cache stats
POST/v1/chat/completions OpenAI-compat chat; supports image_url and softs content items
POST/v1/completions raw-text completions (no chat template applied)
POST/v1/images/prewarm run vision tower + cache softs ahead of time
POST/v1/media/extract run vision, return softs as base64 blob for client-side persistence
GET/v1/kv/snapshot per-session page ownership + refcount (the tenancy viz data source)
GET/v1/cache/stats vision soft-tokens cache hit rate + size
POST/v1/cache/clear flush the vision cache
GET/v1/demo/frames bundled amongus frames as data URLs (dev convenience)
GET/v1/models OpenAI-compat model list
clients are just HTML served from server/static/; server-side state is the engine + vision tower + content-hash prefix cache.
