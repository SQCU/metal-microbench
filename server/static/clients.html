Four bespoke dashboards that render engine internals a generic chat client cannot show: KV-cache page tenancy, batched bandwidth timeseries, vision-cache behaviour, residual-stream steering deltas, tree-of-thoughts branch trees. All LLM work flows through /v1/chat/completions (the OpenAI-compatible surface any other client uses); the engine-state slice that usage.* can't carry — global KV page tenancy, free-pool size, active-stream registry — comes from /v1/engine/state, polled at 1-2 Hz.

tetraplex /static/index.html
Four concurrent chat streams, bandwidth chart + KV tenancy strip. Watch the bandwidth chart ramp from solo decode to a 4× plateau as the engine settles into batched AR at B=4. The tenancy strip colors pages by refcount — bright cells = shared across sessions.
batched AR decode kv tenancy viz multimodal optional
image labeler /static/labeler.html
Drop PNGs (or load the bundled demo frames at /static/demo_frames), apply a schema prompt, watch 4 live-decoding ribbons drain the queue. Vision-cache hit/miss counters update from usage.vision_cache_hits per completion + the global cache snapshot.
vision cache shared schema prefix multi-image queue
loom /static/loom.html
Tree-of-thoughts. Fork N branches from a shared root, each biased by a seed suffix. Click any finished node to fork from there. Sibling branches share their ancestor-path KV pages via the content-hash prefix cache — the global tenancy strip pulses bright while siblings adopt shared pages.
content-hash prefix cache fork-from-node parallel branch decode
residual-stream steering /static/steering.html
Control-vector steering pedagogy. Currently renders a one-shot chat completion with a usage delta-vs-baseline summary; the legacy per-token activation heatmap requires engine-side instrumentation (see the page banner for the deferred-feature pointer).
