# User-agent suggestion interface — design salvage (2026-06)

> Generated by the `user-agent-design-salvage` workflow (12 agents, archaeology over project_archeology.md + thesis docs + the svg_elicit/causal-ophthalmology exemplar + transcripts + current surface + data dirs + git history; 2 independent syntheses reconciled; adversarial completeness critic folded in). ## Thesis

The interface exists to demonstrate user agents made by a varied synthesis mechanism that can run indefinitely to yield more agents, conditioned on the distribution of agents already written AND the distribution of assistants/SCENARIOS presented (26924dc5 @2026-06-01). The deep research target is user-descriptivism: (bio, situation, overlay) tuples whose user-side behavior plausibly samples from the population of actual users engaging chatbots as that population appears in webtext (user_descriptivism_brief.md) — diversity is measured TOWARD that population, not in the abstract; and the harness doubles as a PTQ behavioral instrument (probes that collapse faster as attention/KV projections become less readable over long contexts). The error to correct: every feature has collapsed into a metaphor of an experiment where claude-in-claude-code writes a config file and runs it, with no in-client feedback/iteration; the universal verb became mint-an-experiment-card + spawn lock_in_iterative.mjs + poll-a-read-only-log (index.mjs synthesize-bio-from-coordinates override block + fixed_point.html collectCardFromForm). The SVG/pyrepl harness is the reference exemplar (corpus-driven, API-native, ported into ST-fork toolcards, re-runnable — and its re-run DIVERGENCE is itself a measured pathology, not just a UX nicety). The salvage inverts the verb: expose atomic streaming primitives (designBio/designAgent/selectTargets/judge/split/disambiguate/poll/suggest) as composable, curl-AND-playwright-drivable client verbs whose every result card carries a re-run-with-adjusted-target control; wire the live eff-dim/PCA read-outs into the targeting inputs; restore the thesis-negative regressions (auto-poll-K1 parallel first paint per spec 66; designer.html); de-duplicate the two suggesters + two bio-creators + two eff-dim read-outs into ONE client honoring the four named tenets (compositionality, intent, api-nature, reflexivity-and-automaticity); honor the 7-field orthogonal schema (BIO+MOTIVATION+SCENARIO+RELATIONSHIP+COMM_STYLE+TURN+AUDIT) not a flattened bio-x-agent; purge the surviving Claude-authored fused-name bios (pushy-completionist, gushing-fan); and validate every endpoint against the EXISTING st-debug acceptance suite (60-97).

## Design tenets (19)

### API-nature / P-API-EQUALS-GUI (curl and playwright; rendered artifact or it is not implemented)
Every feature is a real API drivable BOTH by curl into the st-fork driver AND by playwright through a GUI affordance; the ONLY acceptance evidence is a rendered artifact from the real client, validated against the EXISTING numbered st-debug spec suite (specs 60-97 already encode the intended GUI affordance for most features the prior salvage mislabeled config-file-only/unknown). Direct-invoke shims, regex validators, inline ST simulations are NOT validation. Must NOT collapse into claude-writes-a-config-and-runs-it; atomic ops are composable client primitives. CAVEAT: a feature having a spec does not make it pass — many specs (50.E*, 78-fp) are red on unseeded demos.
