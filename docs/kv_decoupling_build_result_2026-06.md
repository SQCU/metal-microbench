## bit_exactness
PASS (with one verified caveat). No hunk perturbs K/V numerics or token bytes. Verified concretely: (1) recordCitation() — the only new clockTick-advancing op on the adoption path — is reachable ONLY via rmapAdd(isCitation:true), which fires ONLY in the adoptSharedPrefixPages adopt loop (lm_engine.swift:1227-1228, cross-session prefix adoption). In the conc1/no-adoption baseline it never fires, so clockTick advances identically to pre-change code, lastAccessTick stamps are identical, and the decayed-citation eviction ties on score==0 and breaks to min(lastAccessTick) = the exact prior LRU. (2) rmapAdd(isCitation:false) does NOT touch clockTick — pure bookkeeping. (3) citationScore/lastCitationTick are NOT in hashSessionKVAtPosition, so LM_KV_OVERWRITE_CHECK is unaffected. (4) Dynamic pool hands out phys indices in the IDENTICAL sequence as the old eager-reverse-init: I traced freeStackPop — old init pushed [base+N-1..base] reversed and popped tail → base, base+1, ...; new growPool appends growFrontier (base, base+1, ...) and allocFresh pops it at the just-appended tail index → same sequence; decref'd pages return via freePush and are reused before growFrontier advances (allocFresh checks free stacks first). (5) always-CoW copyPhysPageKV makes the fresh page byte-identical to the shared source; only the phys index changes (invisible to the kernel and to content-at-position hashing). CAVEAT: this is analytic. The kv_slide_full_geometry memory explicitly warns a single clean run is untrustworthy for this exact corruption class — LM_KV_OVERWRITE_CHECK=1 + a deterministic conc8 multi-run MUST be executed on the Metal box before merge. NOT done (no live engine in this build-only task).

## value_fn_correct
PASS. Citation == distinct ADOPTER, not per-AR-step: recordCitation fires only in the adopt loop, and that loop iterates `beforeAdopted..<match.pages.count` (NEW pages only), with rmapAdd idempotent per (sid,phys) via Set.insert — a session never re-cites its own already-adopted pages, and per-AR self-rereads go through touchAccess (recency), never recordCitation. Decay is correct: decayedCitationScore = storedScore·2^(-(now-lastCitationTick)/halfLife) (KV_CITATION_HALF_LIFE_TICKS default 4096); EWMA update decays-to-now then +1. Eviction picks MIN decayed-citation over the ELIGIBLE set = freeCached (refcount==0); a live wants-slot generation's pages have refcount>0 and are structurally absent from the free stacks → never an eviction victim (no live-set eviction). Tie (all score==0 uncited) breaks to min(lastAccessTick), and touchAccess (B1, preserved in the composed tree — verified it survives the merge) bumps lastAccessTick on every forward-pass block-table write, so the recency tie-break is meaningful. G3 (hot shared prefix accrues citations from distinct adopters → highest value → evicted last), G4 (prefix cited by more distinct adopters than its suffixes → suffix-before-prefix), G5 (decay-to-now ranks a stale-popular prefix low → eventually evicts, no immortality) all fall out. Confirmed against server/test_kv_eviction_guarantees.py G3/G4/G5 thresholds (cache_hits ratios).

## cow_soundness
PASS. Divergent writes are always private by construction (gate retired). The refcount>1 gate that was unsound under the decoupled eviction model (a producer may have decref'd while its trie anchor still exposes the page to a future adopter; two racing adopters both seeing refcount<=1 both skip CoW → the audit #8 clobber) is replaced with unconditional CoW, so soundness no longer depends on any count. rmapSwap/decref retire the shared source safely after the private copy. allocFresh-fail abandons rather than shares. The retained shared half [0..N-1] is read-only (FULL K/V, never divergently written), correctly NOT copied.

## dynamic_pool_safety
PASS. Cannot OOM the 128GB box. Per-layer K/V buffers stay sized at SCRATCH_PAGE_BASE (16000) but are lazy-committed (zero resident RAM until first write via zeroPhysPageKV). The USABLE pool grows on demand lowest-first, one page at a time, capped at poolCap = max(ADMISSION_FREE_PAGE_FLOOR, min(SCRATCH_PAGE_BASE, budgetPages)), budgetPages = floor((KV_MEM_BUDGET_FRAC·physMem − staticResidentBytesEstimate() − 4GB activations)/perPageBytes), perPageBytes computed from actual layer geometry (Σ pg·KV_H·HD·2·2). Default frac 0.90 on 128GB with the ~27GB Q8_0 model yields cap=SCRATCH_PAGE_BASE → identical pool to today (the geometry max is the binding constraint, not the budget). Growth is observable via committedHighWater → stats.committedPages → engine-state JSON. KV_MEM_BUDGET_FRAC env override clamped to (0,1]. RESIDUAL: the cap bounds COMMITTABLE KV; the 4GB activations reserve is flat — a future larger GGUF/bigger activation set must re-verify cap·perPageBytes+model+activations fits resident RAM (open_risk #3, acknowledged).

## admission_cancel
PASS. Sheds WHOLE generations (kill+free), never pause-reprefill. Engine side (submitRequest, lm_engine.swift:2345): bounded loop (shed<8) calls shedLowestValueGeneration which picks the MIN sessionReuseValue wants-slot generation (tie-break least-progressed numGenerated), sets state=.done + doneReason=4 (distinct from 1/EOS, 2/length, 3/error) + errMsg, then closeSession (decref-all + rmap-drop + warm-prefix trie promotion). Never shed .done (already freeing) and never an empty-ownedPages session. No live-page eviction: eligibility is the free list, and shedding frees a whole generation's pages, never evicting a surviving session's working set. Bridge side (server/bridge.py _shed_lowest_value_generation + _resubmit_after_shed_if_backpressure): a SECOND independent tier — on the engine's done_reason==3 'admission backpressure' terminal (ffi_batch.swift:491 errMsg matches bridge .startswith check at 835), sheds the lowest-value NO-active-consumer live generation (protects wire-watched streams) via the existing action=2 path, re-submits under a fresh stream_id, bounded by BRIDGE_ADMISSION_SHED_MAX_ROUNDS. Verified the resubmit's queue-drain is harmless: _consume_engine_stream re-reads _stream_logs from offset 0 and response_q is a notification-only Queue[None] (payload lives in the log), so draining one sentinel loses nothing. The two tiers compose: engine sheds its own low-value gens first; if still refused, bridge sheds a no-consumer zombie. NOTE: doneReason=4 maps in bridge to the !1/!2/!3 fallthrough → finish='stop' (gemma_ffi comment still labels 4 as 'error' — stale comment, harmless since bridge keys on done_reason==3 for the error/backpressure path).
